# Context

Drop any documents here that give the agent useful background about your organisation,
team, or work. There's no required structure or format — anything relevant is fair game.

The agent reads this folder when generating outputs or answering questions. The more
useful context you provide, the better it can tailor roadmap entries and stakeholder
artifacts to your actual situation.

**Examples of things worth adding:**

- An org chart or team list
- A strategy or OKR doc
- A glossary of internal product names or terminology
- Notes on key stakeholders and what they care about
- Known constraints (compliance requirements, hard deadlines, technical debt)
- Previous roadmap decks or board updates

Files can be markdown, plain text, or any format the agent can read. You don't need to
maintain them — just drop them in when they're useful and update them when things change.
